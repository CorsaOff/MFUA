# 🐳 Docker: Полное руководство от А до Я

Этот документ содержит исчерпывающую информацию о Docker: от истории создания до современных практик использования.

---

## 1. Что такое Docker? (Определение)

**Docker** — это платформа с открытым исходным кодом, предназначенная для автоматизации развертывания приложений в легких автономных контейнерах.

**Главная идея:** «Упакуй один раз — запускай где угодно». Контейнер включает в себя всё необходимое для работы: код, системные библиотеки, настройки и среду выполнения.

### Основные понятия:
* **Образ (Image):** "Слепок" или чертеж приложения. Неизменяемый шаблон.
* **Контейнер (Container):** Запущенный экземпляр образа. Изолированный процесс в системе.
* **Движок (Docker Engine):** Клиент-серверное приложение, которое управляет образами и контейнерами.



---

## 2. История появления и развития

### Как это начиналось?
* **2010 год:** Соломон Хайкс запускает компанию **dotCloud** (PaaS-провайдер). Внутри компании назревает потребность в удобном управлении контейнерами.
* **Март 2013 года:** Docker представлен миру как Open Source проект. Это вызвало фурор, так как до этого контейнеризация (LXC) была слишком сложной для обычных разработчиков.

### Этапы эволюции:
1.  **2014 год:** Релиз Docker 1.0. Технология признана стабильной для бизнеса.
2.  **2015 год:** Создание **OCI (Open Container Initiative)**. Docker передает свои стандарты сообществу, чтобы избежать монополии.
3.  **2017 год:** Проект разделяется на **Moby** (для разработчиков систем) и **Docker CE/EE** (для конечных пользователей).
4.  **2020-е:** Docker становится основой для Cloud Native разработки и тесно интегрируется с Kubernetes.

---

## 3. Современное состояние (2024-2026)

Сегодня Docker — это стандарт де-факто. 
* **Dev Containers:** Разработчики пишут код прямо внутри контейнеров, чтобы окружение было идентичным у всей команды.
* **Поддержка Multi-arch:** Один и тот же Dockerfile позволяет собирать образы для процессоров Intel (x86) и Apple/ARM.
* **Безопасность:** Встроенные инструменты сканирования (Docker Scout) находят уязвимости в библиотеках еще до запуска приложения.

---

## 4. Docker Hub и готовые образы

**Docker Hub** — это официальное "облако" (Registry), где хранятся миллионы готовых образов.

### Популярные типы образов:
1.  **Официальные (Official Images):** Поддерживаются командами разработчиков (например, `python`, `nginx`, `postgres`, `ubuntu`).
2.  **Verified Publisher:** Образы от проверенных компаний (например, `redis`, `mongodb`).
3.  **Custom Images:** Образы, созданные сообществом.

**Как это работает:**
* `docker pull nginx` — скачать готовый сервер.
* `docker push my-app` — отправить свой образ в облако.

---

## 5. Почему Docker победил? (Сравнение)
| Характеристика | Виртуальные машины (VM) | Контейнеры (Docker) |
| --- | --- | --- |
| Изоляция | На уровне железа (полная ОС) | На уровне процессов (ядро хоста) |
| Вес | Гигабайты | Мегабайты |
Скорость запуска | Минуты | Секунды
| Производительность | Средняя (накладные расходы на ОС) | Максимальная (почти как нативно) |


## 🎼 Docker Compose — это дирижер
Если приложению нужна база данных и веб-сервер одновременно, используется docker-compose.yml.

YAML

version: '3.8'

services:

web:
  
build: .
    
ports:
    
- "8000:8000"
      
db:
  
image: p
    
ostgres:15
    
environment:
    
POSTGRES_PASSWORD: mysecretpassword
      
## Полезные команды (Cheat Sheet)
docker ps — список запущенных контейнеров.

docker images — список скачанных образов.

docker build -t my-app . — собрать образ из Dockerfile.

docker-compose up -d — запустить проект в фоновом режиме.

## 5. Инструментарий: Dockerfile и Compose

### 📄 Dockerfile — это рецепт
Это текстовый файл, в котором вы пишете инструкции по сборке образа.

```dockerfile
# 1. Базовый слой (ОС + среда)
FROM python:3.10-slim

# 2. Папка приложения внутри контейнера
WORKDIR /app

# 3. Установка зависимостей
COPY requirements.txt .
RUN pip install -r requirements.txt

# 4. Копируем наш код
COPY . .

# 5. Команда запуска
CMD ["python", "main.py"]
